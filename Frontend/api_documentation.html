API Documentation

Welcome to the API documentation for RLG Data and RLG Fans. This documentation provides detailed information on how to interact with our APIs, enabling you to integrate, manage, and utilize our services effectively.

Authentication

All API requests require authentication using an API key. You can obtain your API key from the settings page in your RLG account.

GET /auth/token
Headers: {
    "Authorization": "Bearer YOUR_API_KEY"
}

User Management

Create User

POST /api/users
Headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}
Body: {
    "username": "example_user",
    "email": "example@example.com",
    "password": "securepassword"
}

Get User Details

GET /api/users/{user_id}
Headers: {
    "Authorization": "Bearer YOUR_API_KEY"
}

Social Media Integrations

Fetch Social Media Mentions

GET /api/social/mentions
Headers: {
    "Authorization": "Bearer YOUR_API_KEY"
}
Query Parameters: {
    "platform": "twitter|instagram|facebook|tiktok|linkedin|pinterest|reddit|snapchat|threads",
    "keyword": "search_term",
    "limit": 50
}

Post to Social Media

POST /api/social/post
Headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}
Body: {
    "platform": "twitter",
    "content": "This is a post example.",
    "media_url": "optional_media_url"
}

Data Analytics

Fetch Analytics Data

GET /api/analytics
Headers: {
    "Authorization": "Bearer YOUR_API_KEY"
}
Query Parameters: {
    "date_range": "YYYY-MM-DD to YYYY-MM-DD",
    "metric": "impressions|clicks|engagement"
}

Reporting

Generate Custom Reports

POST /api/reports/generate
Headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}
Body: {
    "report_type": "engagement|performance|seo",
    "parameters": {
        "platform": "twitter",
        "date_range": "YYYY-MM-DD to YYYY-MM-DD"
    }
}

Push Notifications

Send Push Notifications

POST /api/notifications/send
Headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}
Body: {
    "user_id": "user_123",
    "title": "Notification Title",
    "message": "Notification message."
}

Error Codes

All API responses include HTTP status codes to indicate success or failure. Common status codes:

  • 200 OK: Request was successful.
  • 201 Created: Resource was successfully created.
  • 400 Bad Request: Invalid request parameters.
  • 401 Unauthorized: Authentication failed.
  • 500 Internal Server Error: An error occurred on the server.
